The Board — 16 functions of 2 variables

Every function of A and B has exactly one row-pattern below. Find a circuit, submit it, claim the slot. There's room for more than one solution per slot — the board keeps a gallery.

Field guide & extensions

Running the session (~20 min intro)

Introduce AND, OR, NOT and the idea that a wire can fan out — one signal, copied to feed several gates. Build one function together as a group (XOR is a good pick, or one of the NAND/NOR family) and state, without proof, that circuits built from these three gates can realize any Boolean function of any number of variables.

The main event

There are 16 Boolean functions of two variables. Send everyone to the Workbench to hunt for circuits. As each function falls, it gets submitted to the Board. A typical class lands somewhere around 13 of the 16 — a few of the stragglers (looking at you, constant TRUE and constant FALSE) take real cleverness once you're not allowed a free constant gate or a dangling wire.

Scaling up if it's going fast

  • Three variables. There are 256 functions of three variables — nobody needs to find all of them, but ask for a specific handful: majority vote, "exactly one is true," a 1-bit full adder's carry-out.
  • Completeness, for real this time. Ask the class to sketch why AND/OR/NOT can build any truth table — the disjunctive normal form idea (OR together one AND-term per row where the output is 1) falls out naturally once they've been hand-building circuits for an hour.
  • Laws of circuits. De Morgan's laws (NOT(A AND B) = (NOT A) OR (NOT B), and its dual), associativity and distributivity of AND/OR, double negation. Have them verify a law by building both sides and checking the truth tables agree.
  • Minimality. Which of the 16 functions can be built with the fewest gates? Is NAND alone enough for everything? (Yes — that's worth a "why" discussion.)

Using this tool live

The Board is stored per-browser. For a single shared classroom display, keep one browser tab open on the projector's machine and have students call out or walk up to submit — or open the Workbench on their own laptops and just narrate their pattern and gate count to whoever is driving the Board. There's no server behind this, on purpose: it's a static page, easy to host and easy to trust.
